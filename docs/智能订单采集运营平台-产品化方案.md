## 一、现状流程梳理

### 1.1 执行流程总览

当前订单采集流程包含 **6 个主步骤**、**16 个具体操作**，涉及 HBase、HDFS、Flink、BDI、MC 五大系统。

```
┌─────────────────────────────────────────────────────────────────────┐
│  Step 1: 确定源端表结构及 MC 表结构                                   │
│    → 人工比对源端与目标端字段映射                                      │
├─────────────────────────────────────────────────────────────────────┤
│  Step 2: 新建 HBase 表                                              │
│    → 执行建表 jar（指定 ZK 地址/预分区数/压缩方式）                     │
├─────────────────────────────────────────────────────────────────────┤
│  Step 3: 全量初始化（历史全量文件）                                    │
│    → 3.1 创建 HDFS 初始化目录                                        │
│    → 3.2 调整 HDFS 目录权限                                         │
│    → 3.3 上传本地初始化文件到 HDFS                                    │
│    → 3.4 执行 BulkLoad 初始化入库脚本                                 │
├─────────────────────────────────────────────────────────────────────┤
│  Step 4: Flink 增量还原入库                                          │
│    → 4.1 新建 Flink 配置文件（Kafka/HBase/HDFS 参数）                 │
│    → 4.2 在 BDI 上建立采集还原流程                                    │
├─────────────────────────────────────────────────────────────────────┤
│  Step 5: HBase 数据导出到 HDFS                                       │
│    → 5.1 修改提交参数配置文件                                         │
│    → 5.2 新建 Groovy 字段映射脚本                                     │
│    → 5.3 启动导出脚本                                                │
│    → 5.4 验证导出是否成功、字段是否正确                                │
├─────────────────────────────────────────────────────────────────────┤
│  Step 6: HDFS 导出到 MC                                              │
│    → 6.1 建立外表                                                    │
│    → 6.2 建立内表                                                    │
│    → 6.3 执行导出脚本                                                │
│    → 6.4 验证导出推送到 MC 的数据量                                   │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 四大 AI 提效方向

#### 方向一：智能流程自动化（IPA）

```

                │  自然语言需求输入  │
                │  "还原 tf_oh 表"  │

            ▼            ▼            ▼

     │ HBase DDL  │ │ Flink   │ │ Groovy   │
     └────────────┘ └─────────┘ └──────────┘
```

- 自动解析源端表结构，生成 HBase 建表命令（含预分区数智能计算）
- 自动生成 Flink 配置文件（基于表名推断 Kafka Topic、字段映射）
- 自动编排 BDI 流程，无需人工逐步创建
- 一键式全流程执行，跨系统无缝衔接

#### 方向二：智能参数优化

- **预分区数智能计算**：基于初始化文件大小、集群负载、历史经验自动推荐
- **Flink 并行度优化**：根据 Kafka 分区数、数据量、集群资源动态调整 `setParalizem`
- **参数合法性校验**：自动验证 ZK 地址连通性、HDFS 路径存在性、表名规范性
- **模板化配置**：基于表名模式自动填充 90% 参数，仅需确认差异项

#### 方向三：智能监控预警

- **全链路追踪**：BulkLoad → Flink → HBase → HDFS → MC 全程可视化
- **异常实时检测**：Flink 消费延迟、HBase Region 热点、HDFS 空间不足预警
- **自动根因分析**：任务失败时自动定位原因（参数错误/资源不足/网络超时）
- **预测性维护**：基于历史数据预测 Region 分裂时机、磁盘容量告警

#### 方向四：知识图谱赋能

- **经验规则沉淀**：将"分隔符选择规则""预分区计算公式"等编码为可检索知识
- **智能问答**：新手可直接问"tf_oh 表初始化用哪个脚本？"获得精准答案
- **操作推荐**：执行每步操作时，自动推荐相关注意事项和历史故障案例
